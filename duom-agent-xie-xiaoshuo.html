一、单 Agent vs 多 Agent

你平时用 ChatGPT 之类「一个框写全书」，本质上是让 同一个角色又当编剧、又当写手、又当校对。上下文一长，它就开始忘设定、人物前后矛盾。

维度单个 AI 聊天写书多 Agent 协同
角色分工一人包揽所有活世界观/大纲/人设/正文/润色各管一摊
长篇一致性几十章后易失忆跑偏上层约束被锁住，持续对齐
可控性只能靠 prompt 硬掰每层可单独调、互相校验
成品质量越写越平节奏、口吻、结构更稳

二、为什么能解决「长篇跑偏」

跑偏的根因是：设定、大纲、人设这些「上层信息」淹没在几十万字上下文里，单个 AI 顾不过来。

多 Agent 的做法是把这些信息 独立存住、独立管理：

  • 设定层：世界观、力量体系、时间线被锁死，不允许正文改。
  • 大纲层：整书结构和每卷目标清晰，章节不能偏离主线。
  • 人设层：每个角色的语言指纹、动机被固定，不 OOC。
  • 细节层：已发生的关键情节被记录，避免重复或矛盾。

正文 Agent 每次动笔前，都要回头对齐这四层——相当于一直有个人帮你盯住人设和剧情线。

三、橙子AI 的 7 大 Agent 怎么配合

以「四层记忆」为轴，7 大 Agent 在不同层介入：
